# Phase 12 — UI Design Contract

> Visual and interaction contract for Phase 12: Data Layer & Hybrid Builder Removal. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Phase Classification

**This is a data-layer and removal phase.** Phase 12 introduces no new visual components, layouts, or interaction patterns. The UI changes are strictly:

1. **Deletion** of the hybrid builder (dropdowns, slot counter, gate overlay, dcc.Store)
2. **Reuse** of the existing equipment table pattern for the hybrid system
3. **Ungating** the scorecard to render all three columns immediately
4. **Data wiring** of Energy sheet values into existing chart components

All visual output must match the established patterns already used by the mechanical and electrical systems.

## Data Display Contracts

### Hybrid Equipment Table

The hybrid equipment table must be visually identical to the mechanical and electrical equipment tables:

- Same accordion component structure via `make_equipment_section(df, "hybrid", all_data)`
- Same stage grouping order: Water Extraction, Pre-Treatment, Desalination, Post-Treatment, Brine Disposal, Control, Other
- Same columns: Component name, Quantity, Total Cost
- Same cost formatting via `fmt_cost()` from processing.py
- No "preset" label, no banner, no explanatory text (per D-11)

### Energy Data in Charts

- Power breakdown chart: Per-subsystem shaft power bars grouped by system, sourced from Energy sheet rows
- Turbine count chart: Total turbine input / selected turbine size per system (850 kW mechanical, 1500 kW electrical, 850 kW hybrid)
- Chart colors: Existing `STAGE_COLORS` for power breakdown stages, `SYSTEM_COLORS` for per-system series
- Chart layout: No visual changes to chart frame, axis labels, or legend position
